# 90-Day Adoption Roadmap

## Phase 1 (Days 1-14): Foundation

- Finalize `AGENT_RULES.md` derived from Karpathy-style principles.
- Approve legal stance on AGPL/GPL usage boundaries.
- Define evaluation rubric: quality, cost, latency, safety, maintainability.

Exit criteria:

- Rules approved and applied in at least one active repository.
- License risk register signed off.

## Phase 2 (Days 15-45): Core Prototypes

- Prototype memory layer with Cognee on curated docs.
- Prototype context continuity with Claude-Mem in an internal sandbox.
- Integrate Magika into one ingestion path.
- Build one Open Agents-inspired hosted workflow in a non-prod repo.

Exit criteria:

- Measured baseline on retrieval quality, failure modes, and ops cost.

## Phase 3 (Days 46-75): Productization Experiments

- Add voice alert prototype using Voicebox.
- Evaluate GenericAgent and Evolver concepts for internal optimization loops.
- Define benchmark comparison against Kreo-like operational outcomes.

Exit criteria:

- Ranked shortlist of production-candidate components.

## Phase 4 (Days 76-90): Standardization

- Publish Sol-HQ Agent Stack v1 reference architecture.
- Create onboarding docs and reusable templates.
- Define CI checks for prompt/rule quality and ingestion safety.

Exit criteria:

- New projects can bootstrap with standardized templates in under one day.
